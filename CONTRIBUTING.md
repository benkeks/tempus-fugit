# Contributing to Tempus Fugit

## Reporting bugs

We're happy to receive bug reports through the [issue tracker](https://github.com/benkeks/tempus-fugit/issues).
Ideally, provide an explanation of the precise steps and events that lead to the bug, and/or screenshots.

## Workflow

1. Choose an issue from the [issue tracker](https://github.com/benkeks/tempus-fugit/issues).
2. Create a WIP branch to implement the feature or fix.
3. Implement feature.
4. Resolve merge conflicts with `master` (pull continuously to make this easier).
5. Create a [pull request](https://github.com/benkeks/tempus-fugit/pulls) and assign someone of the maintainers to review the code.
6. Iterate until we can merge into `master`.

### Commit message convention:

- Commit messages should be in English.
- Commit message briefly sums up how a commit changes the product. It should work as a continuation of the sentence “This commit will...”
